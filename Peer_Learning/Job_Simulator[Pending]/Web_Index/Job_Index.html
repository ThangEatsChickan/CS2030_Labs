Job Application Simulator

Task Description

This task aim to simulate a simplified naive job application procedure for graduating students. For the sake of simplicity, let's only take into account the students' GPA score and previous internship experience as factors that company consider. 

Level 1

We will first implement the calculation of GPA scores for students. First, define a Gpa class to calculate the overall GPA score given an input of an array of his/her results. You can the grade to GPA mapping at https://gradecalc.info/sg/nus/cumulative_gpa_calc.pl

For example, for the first case ("A","B-"), the GPA is calculated by (5+3)/2 = 4.0

jshell> Gpa.calculate(new String[] {"A","B-"})
==> 4.0
jshell> Gpa.calculate(new String[] {"C","A+"})
==> 3.5
jshell> Gpa.calculate(new String[] {"A","B-","B+","A-","B+"})
==> 4.1
jshell> /exit

Level 2

Now, define a class for Job and Internship.

 Job takes in the name of the role, the overall point requirements, and the vacancies for the role. 

Internship takes in the name of the role and the points it will give the student. 

jshell> new Job("Google Software Engineer", 9.0, 4)
==> 
Full-Time Job: Google Software Engineer
Vacancies: 4
Points Requirements: 9.00
jshell> new Job("DBS Analyst", 8.2, 5)
==> 
Full-Time Job: DBS Analyst
Vacancies: 5
Points Requirements: 8.20
jshell> new Internship("UBS Intern", 2.3)
==> 
Internship: UBS Intern
Points provided: 2.3
jshell> new Internship("Facebook Intern", 3.0)
==> 
Internship: Facebook Intern
Points provided: 3.0
jshell> /exit

Level 3

Now, let's create a class for Student. A student has his/her name, an array of his/her grades, and array of his/her past internships.

The total points for the student is calculated simply by this formula: 

total points = gpa + (sum of all his/her past internships' points) 

jshell> Internship ubsIntern = new Internship("UBS Intern", 2.3)
jshell> Internship fbIntern = new Internship("Facebook Intern", 3.0)
jshell> Internship googleIntern = new Internship("Google Intern" , 4.5)
jshell> new Student("Jack", new String[] {"A","B+","A-"}, 
   ...>             new Internship[] {ubsIntern, fbIntern, googleIntern})
==> 
Name: Jack
GPA: 4.50
Total points: 14.30
jshell> new Student("Jane", new String[] {"A","A+","A"},
   ...>                 new Internship[] {})
==> 
Name: Jane
GPA: 5.00
Total points: 5.00
jshell> /exit

Level 4

Now let's allow the students to apply for jobs and the the jobs would select the students to hire following these rules:

1) The job only select students who has meet the minimum point requirements for the job and then ranks the candidates by their total points (Highest point to lowest point) and select the top n students (where n = vacancies for the role). If there are fewer than n students who applied/qualify for the job, the job can select less than n students to hire. 

2) If a student is already hired by another company, the student won't be considered. 

jshell> Internship ubsIntern = new Internship("UBS Intern", 2.8)
jshell> Internship fbIntern = new Internship("Facebook Intern", 3.0)
jshell> Internship googleIntern = new Internship("Google Intern" , 4.5)
jshell> Student jack = new Student("jack", new String[] {"A","A+","A-"}, 
   ...>             new Internship[] {ubsIntern, googleIntern})
jshell> Student jane = new Student("jane", new String[] {"A","A+","A"},
   ...>                 new Internship[] {ubsIntern})
jshell> Student tom = new Student("tom", new String[] {"A","C+","A","B+"},
   ...>                 new Internship[] {googleIntern})
jshell> Job google = new Job("Google Software Engineer", 9.0, 2)
jshell> Job dbs = new Job("DBS Analyst", 7.0, 2)
jshell> google.apply(new Student[] {jack, jane, tom})
jshell> dbs.apply(new Student[] {jack, jane, tom})
jshell> google.hire()
Hired: 
1: jack
jshell> dbs.hire()
Hired: 
1: tom
2: jane
jshell> /exit

In this case, since Google has already hired Jack, DBS is unable to consider Jack for hiring although he applied for DBS and he has the highest point.


Level 5

The following is a sample run of the program. User input is underlined.

The first integer N represents the number of internships. The next N lines states each Internship name and Internship points. 

The next Integer N represents the number of students. For each student, 

  • The first line is his/her name
  • The second line is his/her grades
  • The third line is his past internships (the index are 0-based and follows the order of internship input. In the example, 0 represents UBS Finance Intern, 1 represents Google Intern, 2 represents JPM Intern)

The next integer N represents the number of jobs. For each job,

  • The first line states the job name, points requirement, and vacancies (In the example, google software engineer role requires 12.5 points minimally and have 2 vacancies)
  • The second line shows the students that applied (the index are 0-based and follows the order of student input. In the example, 0 represents Jack, 1 represent Tom etc etc)

The last line of the input shows the order of hiring, where the index represents the JOBS and are 0-based and follows the order of Job input. In the example, 2 1 0 represents Best Job Ever hiring first, then Credit Suisse Inverstment Banker, then Google Software Engineer. 

The output should show the job and the students that are hired. Note that the job output has to be in order of hiring process stated (in this example, 2 1 0), and in each job, the students hired should be displayed based on their points ranking (Highest point to lowest point).
